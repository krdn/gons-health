안전 데이터 흐름과 3중 게이트

3중 게이트는 합성으로만 안전하다. verified 필터·ABSTAIN 상수·closed-set 어휘 중 하나라도 무너지면 미검증/환각 경고가 약사에게 샌다.

interactions.json→ validateKb()→ lookup()→ ResultCard 하나라도 무너지면 환각 경고가 샌다
1 verified 게이트
src/lib/lookup.ts
verified === true 인 엔트리만 반환. 약사가 1차문헌과 대조한 것만 노출. 나머지는 결과에 절대 안 나옴.
2 abstain 상수
ABSTAIN_MESSAGE
미스 시 문구를 생성하지 않고 고정 상수만 반환. '안전함'을 절대 뜻하지 않음.
3 closed-set 어휘
src/data/vocabulary.ts
입력은 드롭다운 고정 목록뿐. 자유 텍스트 NLP 매칭 원천 배제. KB 키와 정확 일치 필수.

패키징 듀얼 산출물 구조

이 repo는 standalone 웹앱이면서 @krdn/gons-health 코어 패키지다. dist/는 git에 커밋해야 GitHub 의존성 소비처가 깨지지 않는다.

@krdn/gons-health
npm run build → dist/
tsup · React 의존 0인 순수 코어
loadKb / lookup / validateKb / 어휘 / 타입만 export. 다른 프로젝트에 GitHub 태그 의존성으로 임베드.
standalone 웹앱
npm run build:app → dist-app/
tsc + vite · React UI 포함
단독 실행 웹앱. ResultCard 등 UI 컴포넌트는 코어 패키지에서 제외(소비처 React 충돌 방지).
⚠️ 함정: dist/ 는 반드시 git에 커밋. GitHub 의존성은 git-archive 타르볼이라 tracked 파일만 받음 → gitignore하면 소비처가 빈 패키지를 받아 깨짐. 코드 변경 시 dist/ 재빌드 + package.json 함께 커밋.
